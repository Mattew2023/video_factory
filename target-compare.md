# 目标页面判断对比

## 判断依据

本次判断基于你补充的两张截图：

- `current-local-dashboard.png.png`：当前本地 `/dashboard` 页面。
- `reference-qianchuan-dashboard.png`：目标「巨量千川直播大屏 / 全域直播大盘」页面。

结论需要先明确：这两张截图展示的不是同一个页面类型。当前本地页面属于旧版「抖音电商罗盘直播复盘/概览」结构；目标页面属于「巨量千川直播大屏 / 全域直播大盘」结构。两者都与直播数据有关，但信息架构、业务目标、模块组织方式不同。

## 1. 当前本地页面是什么类型？

当前本地 `/dashboard` 是旧版「抖音电商罗盘直播复盘/概览」结构。

从 `current-local-dashboard.png.png` 可以看到：

- 顶部品牌是「抖音电商·罗盘」，页面标题是「直播大屏」，并带「专业版」「直播复盘录屏、评论数...」提示。
- 左侧是罗盘式业务导航，包括「数据」「商品」「人群」「千川」等入口。
- 顶部蓝色指标区展示的是单场直播复盘指标，例如：
  - 直播间成交金额。
  - 直播间用户支付金额。
  - 投放消耗。
  - 千次观看用户支付金额。
  - 成交人数。
  - 成交件数。
  - 观看-成交率、商品点击-成交率、平均在线人数、曝光-观看率、成交老粉占比等。
- 中央主图是「综合趋势」复盘图，上方 tab 包括「综合趋势 / 流量分析 / 主播分析 / 流量诊断 / 引流短视频 / 违规情况」。
- 趋势图下方带有「投放 / 讲解 / 福袋 / 场记 / 预警」等场记事件时间轴。
- 右侧是竖屏直播预览模块，包含「LIVE Dashboard Preview」、播放按钮、已选时间、评论 tab、问题反馈入口等。

这些特征说明当前页的核心目标是「回看并复盘某一场直播」，重点在单场直播结果、直播画面、评论、场记事件和复盘趋势。

因此当前本地页面不是千川全域经营大盘，而是旧版罗盘直播复盘页。

## 2. 目标页面是什么类型？

目标截图 `reference-qianchuan-dashboard.png` 是「巨量千川直播大屏 / 全域直播大盘」结构。

从目标截图可以看到：

- 顶部品牌同时出现「抖音电商」与「巨量千川」，页面语境已经从罗盘复盘转为千川经营大盘。
- 顶部右侧是直播中状态、开播时间、数据口径说明、刷新数据、全屏等实时大盘操作。
- 主指标区强调的是全域经营与投放指标，例如：
  - 整体消耗。
  - 净成交 ROI。
  - 净成交金额。
  - GPM。
  - 观看成交转化率。
  - 实时在线人数。
  - 曝光观看率。
  - 直播间整体观看人数。
- 中央大图是全屏级「整体趋势」图，支持「素材表现」切换、数据口径筛选、分钟粒度、多个经营指标系列。
- 右侧不是直播预览，而是经营分析侧栏：
  - 「成交渠道构成」环形图。
  - 「直播间核心漏斗」转化漏斗。
  - 「直播实时评论」信息流。
- 底部保留「投放动作 / 调控动作」等运营动作标记，服务于投放和调控分析。

这些特征说明目标页面的核心目标是「监控千川投放与全域直播经营效果」，重点在消耗、ROI、成交、渠道构成、转化漏斗、投放动作和实时运营状态。

## 3. 两者不是同一个页面类型

当前本地页面和目标页面不是同一个页面类型。

相同点只是都处在直播数据场景里，也都包含成交金额、消耗、趋势图等指标。但两者的页面定位不同：

- 当前本地 `/dashboard`：旧版「抖音电商罗盘直播复盘/概览」结构，偏单场直播回放、复盘、评论、场记和直播画面。
- 目标页面：新版「巨量千川直播大屏 / 全域直播大盘」结构，偏全域投放经营、渠道拆解、转化漏斗、实时调控和 ROI 监控。

换句话说，当前页回答的是「这场直播复盘表现如何」；目标页回答的是「当前全域直播经营和千川投放跑得如何」。

## 4. 为什么继续微调旧 `/dashboard` 不合理？

继续微调旧 `/dashboard` 不合理，原因不是视觉细节不够，而是页面底层结构不同。

旧 `/dashboard` 的主骨架已经绑定了复盘场景：

- 左侧固定罗盘业务导航。
- 顶部旧版直播复盘指标卡。
- 中央「综合趋势」复盘图。
- 下方场记事件时间轴。
- 右侧竖屏直播预览和评论回放。

这些模块天然服务于「单场直播复盘」。如果继续在这个页面上微调，只能让颜色、字号、圆角、间距逐步接近目标截图，但无法自然形成目标页所需的全域经营结构：

- 目标页需要右侧渠道构成、核心漏斗、实时评论等经营分析卡片，而不是直播预览手机壳。
- 目标页需要以消耗、净成交、ROI、GPM、全域成交数据为核心，而不是以罗盘复盘指标为核心。
- 目标页需要围绕投放动作、调控动作、渠道贡献和转化链路组织，而不是围绕回放、场记、评论和直播画面组织。
- 目标页的整体布局是「顶部状态 + 全域指标总览 + 大趋势图 + 右侧经营分析栏」，不是旧页的「左侧导航 + 中央复盘 + 右侧直播预览」。

因此，继续改旧 `/dashboard` 会变成在旧罗盘复盘页里硬塞千川模块，最终页面会越来越像「披着千川视觉皮的罗盘直播复盘页」，不适合作为目标实现路径。

## 5. 下一步建议

下一步应该新建独立页面：`/qianchuan-dashboard`。

建议保留旧 `/dashboard` 作为罗盘复盘页，不继续把它改成千川大盘。新页面应独立承载目标截图的信息架构，并在路由中新增 `/qianchuan-dashboard` 入口。

推荐的新页面边界：

- `/dashboard`：继续代表旧版抖音电商罗盘直播复盘大屏。
- `/overview`：继续代表旧版罗盘直播概览/列表。
- `/admin`：继续代表当前旧数据配置入口。
- `/qianchuan-dashboard`：新增目标页面，代表巨量千川直播大屏 / 全域直播大盘。

## 6. 旧代码可以复用的部分

旧代码可以复用的是基础能力，不是整页骨架。

建议复用：

- 本地服务 `serve-local.mjs`：现有静态服务已经能支撑新增路由页面。
- 路由判断逻辑：`app.js` 中基于 `window.location.pathname` 切换页面模块的方式可以扩展到 `/qianchuan-dashboard`。
- 数据格式化函数：金额、百分比、数字安全计算等格式化能力可以继续使用。
- canvas 绘图基础：旧趋势图里的 canvas 绘制、坐标换算、hover、tooltip 等基础能力可以作为新图表实现参考。
- localStorage 读写能力：本地保存、读取、导入导出的能力可以继续作为新页面数据源基础。
- 部分颜色变量和深色背景风格：深色大屏背景、蓝色高亮、卡片阴影、按钮状态等视觉基础可以参考。

## 7. 必须新建的部分

为了避免新目标页被旧复盘结构牵制，建议新建以下文件：

- `qianchuanData.js`：承载千川/全域直播大盘的数据模型、默认数据和指标口径。
- `qianchuanDashboard.js`：承载 `/qianchuan-dashboard` 页面渲染、模块组装和交互入口。
- `qianchuanChart.js`：承载目标页趋势图、渠道构成图、漏斗图等图表绘制逻辑。
- `qianchuan.css`：承载目标页独立样式，避免继续污染或受限于旧罗盘复盘页样式。

这些新文件应围绕目标截图重新建模，而不是从旧 `/dashboard` 整体复制后局部替换。

## 8. 不能继续复用的部分

以下旧结构不应继续作为新目标页的主结构：

- 旧顶部指标卡布局：当前指标区是罗盘复盘指标组合，不适合作为千川全域指标总览。
- 旧右侧直播预览模块：目标页右侧应是成交渠道构成、直播间核心漏斗、直播实时评论，而不是竖屏直播预览。
- 旧综合趋势图结构：旧图服务于单场复盘和事件回放，目标图应围绕全域成交、消耗、ROI、渠道和调控动作重新组织。
- 旧 admin 表单结构：旧 admin 面向罗盘复盘数据，不适合作为千川全域大盘的数据配置结构。
- 旧“抖音电商罗盘直播大屏”的整体页面骨架：包括左侧罗盘导航、复盘 tab、直播预览、场记时间轴等整体布局假设。

这里的“不能继续复用”不是要求删除旧代码，而是指不要把它们作为 `/qianchuan-dashboard` 的主实现基础。

## 结论

基于两张截图，可以确认：当前本地页面和目标页面不是同一个页面类型。

当前本地 `/dashboard` 是旧版「抖音电商罗盘直播复盘/概览」结构；目标页面是「巨量千川直播大屏 / 全域直播大盘」结构。继续微调旧 `/dashboard` 不合理，建议停止在旧页面上做结构性仿制，下一步新建独立页面 `/qianchuan-dashboard`。

旧代码可以作为本地服务、路由、格式化、canvas 绘图、localStorage 和部分深色视觉风格的基础素材，但旧顶部指标卡、右侧直播预览、综合趋势图、admin 表单和罗盘直播大屏整体骨架都不应继续复用为新目标页主结构。
