# UI Inventory

本清单用于第三阶段静态布局复刻前确认哪些 UI 可以复刻、哪些只能记录为缺少参考依据。首版只以 `千川页面截图-回放版本.png` 为主视觉基准，不混用罗盘风格。

## 顶部导航区

- 模块名称：顶部导航区
- 依据来源：截图、HTML DOM、CSS
- 固定文案：直播大屏；抖音电商；巨量千川
- 可变数据字段：无。Logo 图片来自 `assets/images/`，不进入业务数据。
- 是否允许进入 `data/dashboard-data.json`：不允许。顶部品牌、标题和 logo 属于锁定 UI。
- 不能自由发挥的 UI 细节：顶部高度、深色背景、左侧品牌 logo 组合、标题位置、分割线、图标尺寸和透明度必须对齐参考图。
- 缺少依据的地方：响应式收缩状态、logo 点击行为、顶部导航 hover/active 态缺少明确截图依据。

## 右上角状态区

- 模块名称：右上角状态区
- 依据来源：截图、HTML DOM、CSS
- 固定文案：开播时间：；数据口径说明；刷新数据；全屏；退出全屏
- 可变数据字段：`liveStatus`、`duration`、`startTime`、`date`
- 是否允许进入 `data/dashboard-data.json`：允许。状态、开播时间、直播时长、日期必须来自数据层。
- 不能自由发挥的 UI 细节：状态文本、时长、开播时间、说明入口、刷新按钮、全屏按钮在同一顶部信息链路中；分隔符、按钮图标、文字颜色、右侧间距不得自行调整。
- 缺少依据的地方：不同直播状态的完整枚举、字段过长时的截断规则、刷新时间反馈、全屏后的状态切换边界缺少依据。

## 主播信息区

- 模块名称：主播信息区
- 依据来源：截图、HTML DOM、CSS
- 固定文案：无固定业务文案
- 可变数据字段：`anchor.name`、`anchor.avatarAsset`
- 是否允许进入 `data/dashboard-data.json`：允许。主播名和头像是业务数据，不允许硬编码在 UI。
- 不能自由发挥的 UI 细节：头像尺寸、圆形裁切、主播名位置、与直播状态的横向关系必须参考截图和 DOM。
- 缺少依据的地方：主播名过长时的省略规则、头像加载失败态、无头像默认图缺少依据。

## 公告/提示条

- 模块名称：公告/提示条
- 依据来源：截图、HTML DOM、CSS
- 固定文案：公告；全域直播大屏全面升级改版，快来体验吧！点击查看详情获取使用手册；查看详情；我知道了
- 可变数据字段：`notice.visible`、`notice.currentIndex`、`notice.total`
- 是否允许进入 `data/dashboard-data.json`：允许。是否展示、当前序号和总数可进入数据层；固定公告文案暂按白名单锁定。
- 不能自由发挥的 UI 细节：公告标签、信息条高度、按钮尺寸、轮播左右箭头、关闭图标、信息条颜色和透明度必须与参考一致。
- 缺少依据的地方：多条公告内容来源、关闭后的持久化规则、按钮点击行为、公告轮播动画缺少依据。

## 核心指标卡区

- 模块名称：核心指标卡区
- 依据来源：截图、HTML DOM、CSS
- 固定文案：整体消耗(元)；净成交ROI；净成交金额(元)；GPM(元)；观看成交转化率；实时在线人数；曝光观看率(次数)；直播间整体观看人数
- 可变数据字段：`topMetrics.totalCost`、`topMetrics.netTransactionRoi`、`topMetrics.netTransactionAmount`、`topMetrics.gpm`、`topMetrics.viewToTransactionRate`、`topMetrics.onlineViewerCount`、`topMetrics.exposureToViewRate`、`topMetrics.liveRoomViewerCount`、`warningCount`
- 是否允许进入 `data/dashboard-data.json`：允许。指标值、百分比、红字提示数量必须来自数据层；指标名称不进入数据层。
- 不能自由发挥的 UI 细节：固定指标顺序、大数字与小数拆分样式、卡片分组、分割线、字体大小、数字对齐方式、红字提示位置不得自由发挥。
- 缺少依据的地方：红字提示完整文案和触发条件、指标 tooltip 内容、指标为空时展示规则缺少依据。

## 趋势图区域

- 模块名称：趋势图区域
- 依据来源：截图、HTML DOM、CSS
- 固定文案：整体趋势；素材表现；综合成本；净成交金额；综合ROI；消耗；整体成交金额；整体支付ROI；每分钟粒度；5分钟粒度；30分钟粒度；投放动作；明细；新建调控；乘方数据；全域净成交数据；全域支付数据
- 可变数据字段：`trendData`、`trendSettings.granularity`、`trendSettings.selectedMetrics`、`trendSettings.availableMetrics`、`warningCount`
- 是否允许进入 `data/dashboard-data.json`：允许。趋势点、时间、指标值、事件数量和选中粒度来自数据层；固定指标标签由 UI 白名单控制。
- 不能自由发挥的 UI 细节：趋势图高度、图例位置、指标切换横排样式、粒度选择器、筛选图标、蓝色/青色折线、tooltip 深色面板、事件标记和“新建调控”入口不得自由发挥。
- 缺少依据的地方：趋势图真实算法、坐标轴刻度规则、tooltip 展示规则、缺失数据插值、筛选器展开态缺少依据。

## 右侧成交渠道卡片

- 模块名称：右侧成交渠道卡片
- 依据来源：截图、HTML DOM、CSS
- 固定文案：成交渠道构成；观看次数；成交金额
- 可变数据字段：`rightCards.channelComposition.activeMetric`、`rightCards.channelComposition.items`
- 是否允许进入 `data/dashboard-data.json`：允许。渠道类目、占比、颜色键和当前指标模式来自数据层；卡片标题和按钮文案锁定。
- 不能自由发挥的 UI 细节：右侧卡片宽度、标题栏、切换按钮、环形图尺寸、图例点、图例顺序、百分比右对齐必须参考截图和 DOM。
- 缺少依据的地方：成交金额模式下的图例值格式、渠道超过一屏时的翻页规则、环形图 tooltip 规则缺少依据。

## 右侧漏斗/转化卡片

- 模块名称：右侧漏斗/转化卡片
- 依据来源：截图、HTML DOM、CSS
- 固定文案：直播间核心漏斗；直播间整体曝光次数；直播间观看次数；商品点击次数；成交订单数
- 可变数据字段：`rightCards.funnel.items`
- 是否允许进入 `data/dashboard-data.json`：允许。每级漏斗数值、比例、条形宽度计算输入来自数据层；阶段名称按白名单锁定。
- 不能自由发挥的 UI 细节：漏斗条层级、深浅蓝色、条形高度、左右偏移、数值位置、画布连接形态不得自行替换成其他图表样式。
- 缺少依据的地方：漏斗比例计算公式、极小值显示规则、空值/零值样式、tooltip 或点击行为缺少依据。

## 右侧评论卡片

- 模块名称：右侧评论卡片
- 依据来源：截图、HTML DOM、CSS、仍缺依据
- 固定文案：直播实时评论；直播画面；暂无评论；主播暂不在播
- 可变数据字段：`comments.state`、`comments.items`、`rightCards.livePanel.mode`、`rightCards.livePanel.coverAsset`
- 是否允许进入 `data/dashboard-data.json`：允许。评论列表、评论状态、直播画面封面和当前模式来自数据层。
- 不能自由发挥的 UI 细节：卡片标题、切换按钮、空状态位置、直播画面背景、遮罩和居中文案需要按参考图执行。
- 缺少依据的地方：有评论状态列表样式、评论头像/昵称/时间规则、滚动行为、直播画面播放态、评论与画面切换后的高度变化缺少依据。

## 底部或辅助信息区

- 模块名称：底部或辅助信息区
- 依据来源：HTML DOM、截图、仍缺依据
- 固定文案：新建调控；投放动作；明细
- 可变数据字段：`trendData[].events`、`warningCount`
- 是否允许进入 `data/dashboard-data.json`：允许。投放动作事件、素材调整次数和红字提示数量来自数据层。
- 不能自由发挥的 UI 细节：辅助入口必须依附在趋势图区域，不新增原网页没有的底部模块；不得加入额外说明、脚注或营销文案。
- 缺少依据的地方：是否存在独立底部栏、红字提示完整样式、动作明细弹窗、调控入口点击后的页面状态缺少依据。
