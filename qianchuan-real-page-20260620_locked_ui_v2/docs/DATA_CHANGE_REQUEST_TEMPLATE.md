# 数据变更请求模板

## 1. 使用说明

这个模板用于提出直播大屏的数据修改需求。用户按模板填写业务数据，Codex 根据填写内容修改：

```text
data/dashboard-data.json
```

这个模板不用于改 UI，不用于改 CSS，不用于改布局，也不用于改截图脚本。

如果只是要改直播间名称、日期、金额、ROI、趋势图、右侧卡片、直播画面状态或最近 7 天数据，请优先使用这个模板。

## 2. 本次允许修改范围

默认只允许修改：

```text
data/dashboard-data.json
```

如果需要替换直播画面图片，必须在请求里明确允许新增一个具体图片文件，并写清楚目标路径。图片只能放在：

```text
assets/live-preview/
```

示例：

```text
允许新增图片文件：assets/live-preview/example.png
```

除非用户明确写出允许修改，其他文件默认不允许修改。

默认不允许修改：

- `AGENTS.md`
- `README_USAGE.md`
- `package.json`
- 页面代码
- CSS
- JS
- 布局
- 截图脚本
- 参考截图
- 源参考项目

## 4. Codex 需要计算或校验

Codex 执行数据修改时，需要计算或校验这些内容：

- `ROI = 净成交金额 / 整体消耗`。
- 百分比字段要统一格式。
- 渠道占比应接近 `100%`。
- 漏斗转化率要符合上下游逻辑。
- 日期、开播时间、趋势图时间、最近 7 天日期不要互相矛盾。
- JSON 结构必须合法。
- 字段名不得擅自改名。
- 不确定的数据要向用户确认，不要猜。

常见校验提醒：

- `topMetrics.viewToTransactionRate`、`topMetrics.exposureToViewRate` 这类字段在 JSON 中写数字，不要带 `%`。
- `rightCards.channelComposition.items[].percent` 写数字，不要带 `%`。
- `rightCards.funnel.items[].ratio` 当前写字符串，通常需要包含 `%`。
- 趋势事件建议只使用 `投放动作` 或 `调控动作`。
- 图片路径从项目根目录开始写，不要写 Windows 绝对路径。

## 5. 不允许修改

数据变更任务中，不允许做这些事：

- 不允许为了改数字去改 UI/CSS/JS。
- 不允许修改截图尺寸。
- 不允许修改截图脚本 URL。
- 不允许修改输出文件名。
- 不允许修改平台固定文案。
- 不允许修改 `rightCards.livePanel.coverAsset` 作为直播画面入口。
- 不允许误用 `shopName`。
- 不允许使用 `git add .`。
- 不允许提交测试数据为默认状态。

特别注意：

- 当前直播间名称字段是 `anchor.name`，不是顶层 `shopName`。
- 直播画面图片入口是 `livePreview.image`。
- `rightCards.livePanel` 当前是保留结构，不作为正式直播画面替换入口。

## 6. Codex 执行前检查

Codex 开始改数据前，应先确认：

- 是否读取 `AGENTS.md`。
- 是否读取 `README_USAGE.md`。
- 是否确认 `git status --short`。
- 是否确认只修改 `data/dashboard-data.json`。
- 是否确认图片新增路径。
- 是否确认不会改 UI。

如果用户请求里没有明确允许新增图片文件，则不要新增图片。

如果工作区不干净，应先说明已有变更，再继续处理。

## 7. Codex 交付后检查

Codex 完成数据修改后，应检查：

- `git status --short`。
- `git diff -- data/dashboard-data.json`。
- JSON 是否合法。
- 是否需要运行 `npm.cmd run screenshot`。
- 截图后是否检查 `output/locked-ui-screenshot.png`。
- 是否需要提交。
- 提交信息建议。

如果用户没有要求运行截图命令，不要主动运行 `npm.cmd run screenshot`。

如果用户没有要求提交，不要主动提交。

提交信息建议格式：

```text
更新直播大屏业务数据
```

如果只替换直播画面图片，可以使用：

```text
更新直播画面图片配置
```
